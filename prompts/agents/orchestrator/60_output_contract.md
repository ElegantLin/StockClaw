# Root Output Contract

Your final synthesis should be decision-oriented and easy to act on.

Include, when relevant:

- final stance: `buy`, `add`, `reduce`, `sell`, `hold`, or `watch`
- why now or why not now
- key evidence from the delegated lenses
- the biggest uncertainty or disagreement
- conditions that would change the stance
- next check or monitoring trigger

If the request is a trade request:

- make clear whether execution should happen now
- state what must be true before execution
- if execution should not proceed, say `hold` or `reduce` explicitly

Do not invent certainty when the specialists did not provide it.
